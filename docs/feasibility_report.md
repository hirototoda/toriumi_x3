# フィージビリティチェック結果レポート

**日付**: 2026-04-16  
**データ**: Community Notes 公式公開データ（ratings 1/8ファイル, 先頭20万行）

---

## 1. 目的

コミュニティノートのステータス変化（Helpful → 削除など）が、ノートの品質ではなく「同一政治クラスタによる集中的な評価（陣営反応）」によって引き起こされているかを検証するパイプラインを構築し、小規模データで試験実行した。

---

## 2. パイプラインの概要

```
Step 1: 評価者に2D座標をつける（polarity）
   ↓
Step 2: 政治トピックのノートに絞る
   ↓
Step 3: ノートごとに「バースト」を検出し、陣営反応 or 自然拡散を判定
   ↓
Step 4: ロジスティック回帰で「陣営反応がノートを潰すか」を統計検定
   ↓
Step 5: 「品質が高いのに陣営に潰されたノート」を特定
```

### 各Stepの詳細

#### Step 1: 評価者に座標をつける

評価者ごとに「どのノートをHELPFUL/NOT_HELPFULと評価したか」のパターンを行列化し、SVD（特異値分解）で2次元に圧縮する。

```
評価行列 R (m人 × nノート):

          note_1  note_2  note_3  ...
rater_A  [ +1     -1      0    ]
rater_B  [ +1     -1     +1    ]
rater_C  [ -1     +1     -1    ]

SVDで2次元に → 各評価者に (polarity_x, polarity_y) 座標がつく
```

- 循環論法を回避するため、各評価者の最初の50件の評価のみで座標を固定
- 座標が近い = 評価の傾向が似ている = 同じ「陣営」

**フィージビリティ確認**: 500人サンプルでシルエットスコア 0.585 → 座標空間に構造がある（分離可能）ことを確認済み。

#### Step 2: 政治トピックに絞る

ノートのテキスト（summary）に政治系キーワード（trump, biden, election, immigration, vaccine 等 40語）を含むかで判定。

- 今回の結果: 全ノートの **23.3%** が政治トピックと判定

#### Step 3: バースト検出 + 分類

あるノートに短時間で評価が集中した区間（＝バースト）を検出し、2種類に分類する。

```
バーストの定義:
  - そのノートの平均評価速度の 3倍以上の速度
  - かつ 5件以上の評価が集中

分類方法（polarity分散で判定）:
  - TypeA（陣営反応）: バースト内の評価者の座標が固まっている（分散 小）
  - TypeB（自然拡散）: バースト内の評価者の座標がバラけている（分散 大）
  - 閾値: 分散の中央値で自動決定
```

#### Step 4: ロジスティック回帰

```
log(p / (1-p)) = β0 + β1·TypeA + β2·TypeB + β3·Trend + β4·Quality

  p        = そのノートが「消された」確率
  β1·TypeA = 陣営反応バーストの影響 ← これが有意なら仮説支持
  β2·TypeB = 自然拡散バーストの影響
  β3·Trend = 評価の時系列推移（統制変数）
  β4·Quality = ノートの品質スコア（統制変数）
```

β3, β4 は「もともと低品質だったから消えた」という反論を排除するための統制変数。

#### Step 5: ターゲット抽出

「品質が高いのに陣営に潰されたノート」を特定する。

```
ターゲット = { 品質Q 上位25% }
           ∩ { TypeAバーストあり }
           ∩ { 最終ステータスが Helpful でない }
```

## 3. 試験実行の結果

### データ規模

| 項目 | 値 |
|------|------|
| ratings 読み込み行数 | 200,000（全体の約0.5%） |
| notes 全量 | 2,580,827 |
| noteStatusHistory 全量 | 2,777,782 |
| polarity算出済み評価者 | 9,149人 |
| 政治トピックのノート（20件以上評価あり） | 385 notes |
| バースト検出 | 385 notes |
| TypeA（陣営反応） | 191 |
| TypeB（自然拡散） | 194 |

### 回帰結果

| 変数 | 係数(β) | p値 | 有意性 |
|------|---------|------|--------|
| 定数項(const) | +2.33 | 0.0000 | *** |
| **TypeA（陣営反応）** | **-0.23** | **0.4071** | **有意でない** |
| TypeB（自然拡散） | +2.57 | 0.0000 | *** |
| Trend（評価推移） | -0.12 | 0.8548 | 有意でない |
| Quality（品質） | -9.96 | 0.0594 | 有意でない（境界） |

### 結果の読み方

- **TypeA (β1) は有意でない (p=0.41)**  
  → このデータでは「陣営反応がノートを潰す」という仮説は支持されなかった

- **TypeB (β2) は有意 (p<0.001)**  
  → 自然拡散バーストがあるノートは消されやすい（ただしこれは「注目されたノートは評価が厳しくなる」という一般的な傾向の可能性）

- **Quality (β4) は境界的 (p=0.059)**  
  → 品質が高いほど消されにくい傾向がわずかに見える。全データなら有意になる可能性あり

### ターゲットノート

- 46件のノートが「品質上位25% かつ TypeAバースト かつ 非Helpful」に該当

---

## 4. 注意点・制限

1. **データが全体の約0.5%**: ratings は 8分割ファイルの1つの先頭20万行のみ。全データでは結果が変わる可能性が高い
2. **ratings と notes/history のファイル番号が異なる**: ratings-00003 と notes-00000/history-00000 を使用しており、noteId の重複が部分的
3. **政治トピック判定が粗い**: キーワードマッチのみ。より精密な分類（LLM等）を検討する余地あり
4. **品質スコアQが簡易**: URL数 + 文字数のみ。ドメイン信頼性等の追加指標で改善可能

---

## 5. 次のステップ

1. **全データでの再実行**: ratings 全8ファイルを使用する
2. **トピックの絞り込み**: 「政治」全体ではなく特定のトピック（例: ワクチン、選挙、移民等）に絞ることで陣営反応が検出されやすくなる可能性
3. **品質スコアの精緻化**: URL数 + 文字数 → ドメイン信頼性を追加
4. **200件ラベル付け**: 品質スコアQの精度を人手/LLMで検証
