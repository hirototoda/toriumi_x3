---

## p1: タイトル・イントロ

コミュニティノート「信者バトル」仮説検証。ノートの品質ではなく陣営反応がステータスを決めているのではないか。

## p2: 仮説と検証方針

ステータス変化が陣営反応（同一クラスタの集中評価）に駆動されているか、5ステップで検証する。

## p3: データ概要（担当: Sさん）

公式公開TSV 3ファイル（notes / ratings / noteStatusHistory）をnoteIdで結合。使用カラムと型の説明。

## p4: Step1 前処理・polarity計算（担当: Sさん）

bridgingアルゴリズム再実装。各評価者の最初の50件でpolarity固定（循環論法回避）。総評価数20件以上に絞り込み。

## p5: Step2 トピック分類（担当: Gさん）

政治トピックのノートのみ抽出。APIコスト$0を目標とした手法で分類。

## p6: Step3 バースト検出・分類（担当: Gさん）

バースト = 平均評価速度の3倍以上 かつ 5件以上。TypeA（陣営反応: polarity偏り大）とTypeB（自然拡散: polarity分散大）に分類。

## p7: Step4 ロジスティック回帰（担当: 自分）

log(p/(1-p)) = β0 + β1·TypeA + β2·TypeB + β3·Trend + β4·Quality。統制変数で「低品質だから消えた」を排除。

## p8: Step5 品質スコアと「正しいのに消えた」ノート（担当: 自分）

Q = URL数 + ドメイン信頼性 + 文字数（正規化）。ターゲット = {Q上位25%} ∩ {TypeAバースト} ∩ {非Helpful}。

## p9: 結論と判断基準（担当: 自分）

TypeAのみ有意 → 仮説支持。TypeA・B両方有意 → 自然拡散主因の可能性。どちらも無意 → 別要因。
