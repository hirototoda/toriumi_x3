## やったことの手順

### 手順1: 評価者の「立場」を数値化した

Community Notes の評価データ（ratings.tsv）には「誰が、どのノートを、HELPFUL/NOT_HELPFUL と評価したか」が記録されている。

この評価パターンを行列にして、SVD（特異値分解）で2次元に圧縮した。

```
例: 3人の評価者が3つのノートを評価

              note_1  note_2  note_3
評価者A      [  +1     -1      +1  ]    ← AとBは似た評価パターン
評価者B      [  +1     -1      +1  ]
評価者C      [  -1     +1      -1  ]    ← Cは正反対

SVDで2次元に圧縮 →

評価者A: (0.8, 0.1)  ─┐ 座標が近い = 同じ「陣営」
評価者B: (0.7, 0.2)  ─┘
評価者C: (-0.9, -0.1)    座標が遠い = 違う「陣営」
```

- 循環論法を防ぐため、各評価者の**最初の50件の評価だけ**で座標を固定
- この座標で「陣営」を事前に決めるのではなく、後のステップで連続値のまま使う

**検証結果**: 500人をサンプリングしてプロットしたところ、シルエットスコア 0.585 で**評価空間に構造がある（立場の違う集団が存在する）**ことを確認。

### 手順2: 政治系ノートに絞った

ノートの本文に政治系キーワード（trump, vaccine, immigration 等 40語）を含むかで判定。

- 結果: 全ノートの **23.3%** が政治トピック

### 手順3: ノートへの評価が短時間で集中した区間（バースト）を検出し、分類した

```
バーストの定義:
  そのノートの平均評価速度の 3倍以上 かつ 5件以上が集中

バーストの分類（評価者の座標の分散で判定）:
  TypeA（陣営反応）: 座標が固まっている → 同じ陣営が集団で評価
  TypeB（自然拡散）: 座標がバラバラ   → いろんな人が自然に集まった
```

### 手順4: ロジスティック回帰で統計検定した

```
log(p / (1-p)) = β0 + β1·TypeA + β2·TypeB + β3·Trend + β4·Quality

  p    = そのノートが NOT_HELPFUL になった確率
  β1   = 陣営反応(TypeA)の影響 ← これが有意なら仮説を支持
  β3,β4 = 統制変数（「もともと低品質だから消えた」という反論を排除するため）
```

### 手順5: 「品質が高いのに潰されたノート」を特定した

```
ターゲット = 品質スコア上位25% ∩ TypeAバーストあり ∩ Helpfulになれなかった
```

## 結果の正直な解釈

### 良い点

1. **パイプラインが動く**: Step 1〜5 まで一気通貫で実行でき、結果が出る
2. **評価空間に構造がある**: 2Dプロットで陣営の分離を確認済み
3. **TypeA が有意に出た (p=0.005)**: 統計的には有意

### 注意すべき点

1. **n=43 は少ない**
   - ロジスティック回帰では説明変数1つにつき最低10〜20件が必要
   - 今回は説明変数4つに対して43件 → ギリギリ。結果が不安定な可能性

2. **β の符号が仮説と逆**
   - 仮説: TypeA → ノートが潰される → β は正のはず
   - 結果: TypeA → β = **-1.43（負）** → TypeAがあるとむしろ**潰されにくい**
   - 解釈: 同じ陣営が集中的に「Helpful」と評価するケースも TypeA に含まれるため

3. **データが全体の0.5%**
   - 全データで結果が変わる可能性が高い
   - 特にトピック別分析は全データでないと成立しない
